# ArUco Indoor Localization System for Mobile Robotics (ROS 2 Humble)

Система локализации мобильного робота внутри помещений на основе потолочных ArUco-маркеров и монокулярной веб-камеры (Logitech C270). Проект разработан для среды **ROS 2 Humble** на базе **Ubuntu 22.04 LTS** (поддерживается как физическая система, так и виртуальное окружение WSL2 под Windows 11).

Алгоритм вычисляет точное 3D-положение робота ($X, Y, Z$) и его пространственную ориентацию (в виде кватернионов) относительно глобальной карты маркеров и публикует данные в стандартный топик ROS 2.

## 🛠 Требования и зависимости

### 1. Операционная система и окружение
* **OS:** Ubuntu 22.04 LTS (Jammy Jellyfish)
* **ROS 2:** Humble Hawksbill ([Официальное руководство по установке](https://docs.ros.org/en/humble/Installation.html))

### 2. Зависимости Python
Для работы модулей компьютерного зрения необходимы расширенные пакеты OpenCV и утилиты работы с матрицами. Установите их в терминале Ubuntu:

```bash
sudo apt update
sudo apt install python3-pip python3-opencv
pip3 install numpy pyyaml opencv-contrib-python
```

⚠️ Важно: Пакет opencv-contrib-python обязателен, так как модули aruco для поиска маркеров находятся именно в репозитории расширений (contrib).

## 📷 Настройка оборудования (Камера)

Система захвата видео использует бэкенд **V4L2** и принудительно кодирует поток в формат MJPEG с разрешением **640x480**. Это существенно снижает нагрузку на шину данных и полностью устраняет задержки захвата кадров в виртуальных средах (**select() timeout**).

### Вариант А: Чистая Ubuntu 22.04

Если Ubuntu установлена как основная ОС, вам нужно лишь предоставить текущему пользователю права на доступ к видеоустройствам. Выполните в терминале:

```bash
sudo chmod 666 /dev/video*
```

### Вариант Б: WSL2 (Ubuntu 22.04 внутри Windows)

Поскольку среда WSL2 изолирована, необходимо принудительно пробросить USB-камеру из хост-системы Windows с помощью утилиты **usbipd-win** версии 5.x+.

**Этап 1. В Windows:** Скачайте и установите последнюю версию [usbipd-win](https://github.com/dorssel/usbipd-win/releases).

**Этап 2. В Windows (PowerShell от Администратора):** 
1. Сначала откройте ваш обычный терминал Ubuntu (чтобы запустить подсистему WSL2 в фоне).  
2. В окне PowerShell проверьте список подключенных к компьютеру USB-устройств:

```powershell
usbipd list
```
3. Найдите в списке вашу веб-камеру (**Logitech, Inc. Webcam C270**) и запомните её BUSID (например, **2-2**).
4. Привяжите и пробросьте устройство в WSL:

```PowerShell
usbipd attach --wsl --busid 2-2
```
(Оставьте это окно PowerShell открытым на время работы проекта).

В терминале WSL2 (Ubuntu): Предоставьте права доступа к созданному виртуальному файлу видеопотока:

```Bash
sudo chmod 666 /dev/video*
```

### Шаг 1: Калибровка камеры
Чтобы алгоритм **solvePnP** правильно определял расстояние в метрах, камере необходимо компенсировать оптические искажения (дисторсию) линзы.

1. Распечатайте калибровочную шахматную доску (паттерн с сеткой 9х6 внутренних углов) на листе А4 и зафиксируйте её на твердой плоскости (картон или планшет).

2. Измерьте сторону одного квадрата линейкой в метрах и укажите её в переменной **SQUARE_SIZE** в файле **calibrate.py** (например, **0.025** для 2.5 см).

3. Запустите скрипт калибровки:

```Bash
python3 calibrate.py
```

4. Показывайте доску камере под разными углами и наклонами, старась перекрывать разные части кадра. Когда сетка углов успешно подсветится цветными линиями, нажимайте Пробел, чтобы сохранить кадр.

<img width="1280" height="758" alt="image" src="https://github.com/user-attachments/assets/ddae66da-e166-4853-b085-46e9ef5b0000" />

5. Собрав 20–35 успешных кадров, нажмите клавишу **q**. Скрипт рассчитает коэффициенты и автоматически сгенерирует файл **calibration.yaml**.

### Шаг 2: Локальное тестирование детекции

Перед расчетом навигации убедитесь, что OpenCV корректно видит камеру и распознает ваши маркеры. Запустите тест:

```Bash
python3 test_cam.py
```

Направьте камеру на распечатанный потолочный маркер. Вокруг него должна появиться зеленая рамка с индикацией его реального аппаратного ID (в данном проекте используются маркеры с ID 1 и ID 2).

### Шаг 3: Тестирование локализации на чистом Python

Запустите промежуточный математический прототип без участия ROS-инфраструктуры, чтобы проверить правильность расчетов:

```Bash
python3 localize_test.py
```

Скрипт выведет окно трансляции, нарисует на маркере виртуальные 3D-оси координат (RGB) и отобразит в консоли расстояние. Убедитесь, что координата $Z$ адекватно реагирует на приближение и удаление камеры от метки.

### Шаг 4: Запуск ROS 2 ноды локализации
Сделайте файл ноды исполняемым в вашей системе:

```Bash
chmod +x aruco_locator_node.py
```

Откройте первый терминал, активируйте ROS 2 окружение и запустите ноду:

```Bash
source /opt/ros/humble/setup.bash
python3 aruco_locator_node.py
```

Откройте второй терминал Ubuntu для прослушивания публикуемого топика локализации:

```Bash
source /opt/ros/humble/setup.bash
ros2 topic echo /robot_pose
```

В консоли второго терминала начнет непрерывно публиковаться текущая пространственная поза робота.

### ⚙ Конфигурация параметров карты
Если вы изменяете геометрию расположения меток на потолке, их количество или физические размеры, отредактируйте параметры в блоке инициализации __init__ внутри файла **aruco_locator_node.py**:

**self.marker_size** — Физический размер стороны черного квадрата маркера в метрах (в данном проекте: 0.270 для 270 мм).

**self.aruco_dict** — Используемый словарь семейства ArUco (в данном проекте: **cv2.aruco.DICT_7X7_250**).

**self.marker_world_map** — Глобальная база данных положения меток на потолке в формате **{ID: np.array([X, Y, Z_потолка])}**.

Текущая конфигурация проекта под 2 маркера:

```Python
self.marker_world_map = {
    1: np.array([0.0, 0.0, 2.71]),    # Маркер ID 1 — начало координат (0,0), высота потолка 2.71 м
    2: np.array([1.183, 0.0, 2.71])   # Маркер ID 2 — смещен строго по оси X на 1.183 м от первого
}
```

### 📊 Структура ROS 2 сообщения

Данные о положении мобильного робота публикуются в топик /robot_pose с использованием стандартного типа сообщения geometry_msgs/msg/PoseStamped:

```YAML
header:
  stamp:
    sec: 1781201218
    nanosec: 178537936
  frame_id: map
pose:
  position:
    x: 0.45231940
    y: -0.0210543
    z: 1.63124472
  orientation:
    x: 0.79142128
    y: -0.5904483
    z: -0.1056639
    w: 0.11772087
```
